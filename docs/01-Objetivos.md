# Objetivos

## Objetivo general

Desarrollar dos APIs desacopladas para realizar operaciones de tipo de cambio, validar el usuario, registrar la solicitud y persistir la informacion requerida.

## Objetivos especificos

- Exponer API Experience para la comunicacion con el cliente.
- Implementar autenticacion mediante JWT.
- Validar mediante GoREST que el usuario exista y obtener su nombre.
- Exponer API Support para registrar, actualizar y buscar el tipo de cambio.
- Registrar quien realizo cada solicitud.
- Persistir nombre de usuario, monto inicial, monto final, tipo de cambio y fecha del proceso.
- Utilizar Java, Spring Boot y programacion reactiva.
- Documentar las APIs mediante Swagger.
